# ADR 0002: Use React and Go

## Status

Accepted

## Context

The project needs a modern frontend and a backend suitable for API development.

## Decision

Use React with TypeScript for the frontend and Go for the backend.

## Consequences

### Positive

- React is widely used in frontend development
- TypeScript improves maintainability
- Go is strong for APIs and cloud-native services
- The stack demonstrates full-stack capability

### Negative

- Requires maintaining two language ecosystems
